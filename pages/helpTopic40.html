    How does A Better Finder Rename exploit the EXIF meta information from my digital camera pictures?
 

A Better Finder Rename currently only exploits the creation date and time of JPEG/EXIF and RAW pictures.

This is particularly useful for:

  • adding the date and time a picture was taken to the file name, e.g. "my picture 12/01/2004 12:00:01"
  • renaming the picture file to the time and date it was taken, e.g. "12/01/2004 12:00:01"
  • creating numbered list where the sorting order corresponds to the actual order in which the pictures were taken rather than relating only to the creation time of the file, e.g. image1.jpg, image2.jpg, ...
The renaming actions that currently support EXIF dates are:

Sequence Numbers
  • Produce sequence number list
  • Add sequence number at beginning
  • Add sequence number to end
  • Change existing sequence numbers
  • Insert sequence number in front of existing text
  • Insert sequence number after existing text
  • Insert sequence number at position
Date & Time
  • Rename to date/time
  • Add date/time to beginning
  • Add date/time to end
Alphabetical Sequences
  • Produce alphabetical list
  • Add alphabetical sequence at beginning
  • Add alphabetical sequence to end
Image Dimensions
  • Add image dimensions to end
  • Add image dimensions to beginning
  • Rename to image dimensions
Advanced & Special
  • Rename from file list
Please Note: If a file does not contain any EXIF information, its creation date will be used for sorting.
